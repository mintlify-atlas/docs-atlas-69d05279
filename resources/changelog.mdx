## Version History Highlights

### Recent Updates

Moving icons is continuously evolving with regular updates that include:

- New animated icon additions based on Lucide icon releases
- Performance improvements for smoother animations
- Enhanced TypeScript support
- Better tree-shaking for smaller bundle sizes
- Improved documentation and examples

### Major Milestones

<AccordionGroup>
  <Accordion title="530+ Icons">
    The library has grown to include 530+ hand-crafted animated icons, covering a wide range of use cases from UI elements to specialized graphics.
  </Accordion>
  
  <Accordion title="Svelte 5 Native">
    Built natively for Svelte 5 with full support for runes, snippets, and the latest Svelte features.
  </Accordion>
  
  <Accordion title="Zero Dependencies">
    Maintained as a zero-dependency library, ensuring minimal impact on your bundle size and no version conflicts.
  </Accordion>
  
  <Accordion title="shadcn-svelte Integration">
    Added registry support for seamless integration with shadcn-svelte, allowing you to add icons via CLI.
  </Accordion>
</AccordionGroup>

## Migration Guides

When upgrading between major versions, check the release notes for migration guides and breaking changes.

### Checking Your Version

To see which version you're currently using:

```bash
npm list @jis3r/icons
```

### Updating to Latest Version

To update to the latest version:

```bash
npm update @jis3r/icons
```

Or to install a specific version:

```bash
npm install @jis3r/icons@2.5.0
```

## Reporting Issues

Found a bug or have a suggestion?

<Steps>
  <Step title="Search existing issues">
    Check if the issue has already been reported on [GitHub Issues](https://github.com/jis3r/icons/issues).
  </Step>
  
  <Step title="Create detailed report">
    If it's a new issue, create a detailed bug report including:
    - Your version number
    - Steps to reproduce
    - Expected vs actual behavior
    - Browser/environment details
  </Step>
  
  <Step title="Submit feature requests">
    For new features or improvements, open a discussion or feature request with your use case and proposed solution.
  </Step>
</Steps>
