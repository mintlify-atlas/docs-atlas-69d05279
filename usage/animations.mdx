## Default Behavior

By default, all Moving Icons animate on hover. Each icon has a unique, hand-crafted animation that plays when you hover over it:

```svelte
<script>
  import { Bell, Star, Activity } from '@jis3r/icons';
</script>

<!-- These icons will animate on hover -->
<Bell />
<Star />
<Activity />
```

<Note>
  Hover animations are built into each icon component. No additional configuration is needed for basic hover behavior.
</Note>

## How Animations Work

Each icon component includes:

1. **A hover handler** that triggers the animation
2. **Vanilla CSS animations** using `@keyframes`
3. **An automatic reset** after the animation completes

Here's how it works internally (from `/home/daytona/workspace/source/src/lib/icons/bell.svelte:12-18`):

```svelte
function handleMouseEnter() {
  if (animate) return;
  animate = true;
  setTimeout(() => {
    animate = false;
  }, 1100);  // Duration matches animation length
}
```

<Tip>
  Animations are implemented with pure CSS `@keyframes` and CSS classes - no JavaScript animation libraries required. This ensures optimal performance.
</Tip>

## Animation Timing

Different icons have different animation durations based on their complexity:

- **Simple animations**: 600ms (e.g., Star)
- **Medium animations**: 900-1100ms (e.g., Bell)
- **Complex animations**: 1400ms (e.g., Accessibility)

The `setTimeout` in each icon matches its animation duration to ensure smooth resets.

## Controlled Animations

You can take full control of animations using the `animate` prop:

### Basic Control

```svelte
<script>
  import { Bell } from '@jis3r/icons';
  
  let animate = $state(false);
  
  function triggerAnimation() {
    animate = true;
    // Manually reset after animation completes
    setTimeout(() => {
      animate = false;
    }, 1100);
  }
</script>

<button onclick={triggerAnimation}>
  <Bell {animate} />
  Trigger Animation
</button>
```

### Hover Control from Parent

Control animations from a parent element's hover state:

```svelte
<script>
  import { Bell } from '@jis3r/icons';
  
  let animate = $state(false);
</script>

<button
  onmouseenter={() => animate = true}
  onmouseleave={() => animate = false}
  class="flex items-center gap-2 px-4 py-2"
>
  <Bell size={16} {animate} />
  <span>Notifications</span>
</button>
```

<Warning>
  When `animate` is set to `true` explicitly, the icon's built-in hover handler returns early and does nothing. This prevents conflicting animation triggers.
</Warning>

## Animation Implementation

Animations use vanilla CSS with class-based triggers. Here's an example from the Bell icon:

<CodeGroup>
```svelte Component Structure
<svg
  class:animate-svg={animate}
>
  <path class:animate-bell={animate} />
  <path class:animate-clapper={animate} />
</svg>
```

```css CSS Animations
.animate-svg {
  transform-origin: top center;
  animation: bellRing 0.9s ease-in-out;
}

.animate-bell {
  animation: bellMove 1.1s ease-in-out;
}

.animate-clapper {
  animation: clapperMove 1.1s ease-in-out;
}

@keyframes bellRing {
  0% { transform: rotate(0deg); }
  16.67% { transform: rotate(20deg); }
  33.33% { transform: rotate(-10deg); }
  50% { transform: rotate(10deg); }
  66.67% { transform: rotate(-5deg); }
  83.33% { transform: rotate(3deg); }
  100% { transform: rotate(0deg); }
}
```
</CodeGroup>

## Animation Patterns

Moving Icons use various CSS animation techniques:

### Transforms

```css
/* Rotation */
transform: rotate(360deg);

/* Scaling */
transform: scale(1.2);

/* Translation */
transform: translateX(-6px);
```

### Transform Origins

Animations use precise transform origins for natural movement:

```css
/* From accessibility.svelte */
.accessibility-group2 {
  transform-origin: 9px 16px;  /* Specific pivot point */
  transition: transform 0.3s ease;
}
```

### Staggered Animations

Multiple elements animate in sequence using animation delays:

```css
/* From accessibility.svelte */
.animate .group1 {
  animation: group1Rotate 0.8s ease-in-out;
}

.animate .group2 {
  animation: group2Rotate 1s ease-in-out 0.4s;  /* 0.4s delay */
}

.animate .path3 {
  animation: path3Rotate 0.4s ease-in-out 0.2s;  /* 0.2s delay */
}
```

## Advanced Use Cases

### Toggle Animation on Click

```svelte
<script>
  import { Star } from '@jis3r/icons';
  
  let isActive = $state(false);
  let animate = $state(false);
  
  function handleClick() {
    isActive = !isActive;
    
    // Trigger animation
    animate = true;
    setTimeout(() => {
      animate = false;
    }, 600);
  }
</script>

<button onclick={handleClick}>
  <Star 
    {animate}
    color={isActive ? 'gold' : 'currentColor'}
  />
  {isActive ? 'Starred' : 'Star this'}
</button>
```

### Animate on Data Change

```svelte
<script>
  import { Bell } from '@jis3r/icons';
  
  let notifications = $state(0);
  let animate = $state(false);
  
  $effect(() => {
    // Animate when notifications change
    if (notifications > 0) {
      animate = true;
      setTimeout(() => {
        animate = false;
      }, 1100);
    }
  });
</script>

<button onclick={() => notifications++}>
  <Bell {animate} />
  {#if notifications > 0}
    <span class="badge">{notifications}</span>
  {/if}
</button>
```

### Loop Animation

```svelte
<script>
  import { Activity } from '@jis3r/icons';
  import { onMount } from 'svelte';
  
  let animate = $state(false);
  
  onMount(() => {
    // Loop animation every 3 seconds
    const interval = setInterval(() => {
      animate = true;
      setTimeout(() => {
        animate = false;
      }, 600);
    }, 3000);
    
    return () => clearInterval(interval);
  });
</script>

<Activity {animate} />
<span>Loading...</span>
```

### Programmatic Trigger

```svelte
<script>
  import { Bell } from '@jis3r/icons';
  
  let animate = $state(false);
  
  export function playAnimation() {
    animate = true;
    setTimeout(() => {
      animate = false;
    }, 1100);
  }
</script>

<Bell {animate} />
```

## Performance Considerations

<Tip>
  Moving Icons use CSS animations instead of JavaScript-based animations for optimal performance. CSS animations:
  
  - Run on the GPU when possible
  - Don't block the main thread
  - Have smooth 60fps playback
  - Are hardware-accelerated
</Tip>

### Best Practices

1. **Avoid animating too many icons simultaneously** - Stagger animations if animating multiple icons
2. **Use `transform` and `opacity`** - These properties are GPU-accelerated
3. **Respect user preferences** - Consider `prefers-reduced-motion`:

```svelte
<script>
  import { Bell } from '@jis3r/icons';
  
  let prefersReducedMotion = $state(false);
  
  $effect(() => {
    const mediaQuery = window.matchMedia('(prefers-reduced-motion: reduce)');
    prefersReducedMotion = mediaQuery.matches;
  });
  
  let shouldAnimate = $state(false);
  let animate = $derived(prefersReducedMotion ? false : shouldAnimate);
</script>

<button
  onmouseenter={() => shouldAnimate = true}
  onmouseleave={() => shouldAnimate = false}
>
  <Bell {animate} />
</button>
```
