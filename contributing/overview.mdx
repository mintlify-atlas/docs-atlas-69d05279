## Pull Request Guidelines

When submitting a pull request, please follow these guidelines:

<Steps>
  <Step title="Make descriptive commit messages">
    Include as much information as you can. Explain anything that the file diffs themselves won't make apparent.
  </Step>
  
  <Step title="Document your pull request">
    Explain your fix, link to the relevant issue, and add screenshots when adding new icons.
  </Step>
  
  <Step title="Target the correct branch">
    Most bugfixes or new features should go to the `main` branch.
  </Step>
  
  <Step title="Keep it focused">
    Include only related work. Pull requests with unrelated commits won't be accepted.
  </Step>
</Steps>
